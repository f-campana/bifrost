01Overview

Sanctum is the compiled knowledge wiki. It owns everything the system knows — raw sources, synthesis, memory, indexes, project context. Knowledge compounds here over time.

Bifrost is the schema and operating system that maintains the wiki. It owns skills, contracts, routing rules, verification logic, and prompt assembly rules. It tells agents how to operate.

Projects consume both. The orchestrator is a human. Automation is earned, not assumed.

Bifrost is not the knowledge base. It is the operating system for the knowledge base.
When in doubt: does it define how the wiki is maintained? → Bifrost. Does it contain wiki content? → Sanctum.
The Karpathy Model

This system is a direct implementation of Karpathy's LLM Wiki pattern, generalised into a broader agent OS.

Three-layer architecture
Raw Sources
Immutable inputs. LLM reads, never modifies. Provenance preserved permanently.
→ Sanctum 40 Sources/
Compiled Wiki
LLM-maintained synthesis. Compounds over time. Good answers become wiki pages.
→ Sanctum 10 Knowledge/ + 01 Indexes/ + 30 Memory/
Schema
Operating instructions. Defines ingest, query, lint. Skills, contracts, routing.
→ Bifrost
02Core Principles
Human orchestrator
The human makes all routing, decomposition, and promote gate decisions. No agent decides which agent runs next. Automation is for execution, not orchestration.
Sources are immutable
Raw source material in 40 Sources/ is never modified by agents. LLM reads from it but writes only to the compiled wiki zone. Provenance must be traceable.
The wiki compounds
Every useful synthesis — an answer, an analysis, a discovered connection — gets filed back into the wiki. Nothing valuable disappears into chat history. The system gets smarter with every interaction.
Bifrost is the schema
Bifrost defines how the wiki is maintained. Sanctum contains what the wiki knows. Different concerns that must never blur.
Harness agnosticism
AGENTS.md and SKILL.md are the harness contract. Pi, Claude Code, and Codex all read them. Skills discovered via ~/.agents/skills/ standard global path.
Pull before push
Every automated trigger starts as manual dispatch. ≥2 weeks predictable manual use before any automation. Automation is a reward for proven reliability.
Dogfood the build
Each phase uses what was built before. docs/changelog writes its own changelog. wiki/ingest files the notes from its own creation session.
Promote gate
Nothing moves from agent output to durable wiki without human review. Background agents write to 00 Inbox/ only.
Narrow context
Action agents: ≤10 files. Workflow agents: one domain map. Project agents: cross-domain maps. Strategy (you): full context. Loading more than needed causes scope creep.

05Bifrost — The Schema Layer

Bifrost is a public repository. It defines how agents maintain Sanctum and execute tasks. Skills are discovered via ~/.agents/skills/bifrost/ — symlinked from bifrost/skills/. No custom registry needed; native harness discovery handles it.

Folder structure
bifrost/ README.md · AGENTS.md · system-spec.md skills/ _index.md · _template/SKILL.md wiki/ wiki-ingest/ · wiki-query/ · wiki-lint/ docs/ docs-changelog/ · docs-write/ · docs-drift/ · docs-adr/ react/ react-implement/ · react-test/ · react-review/ react-bug-find/ · react-perf-audit/ typescript/ ts-type/ · ts-audit/ express/ express-route/ · express-schema/ github/ gh-read-file/ · gh-read-diff/ · gh-create-branch/ gh-open-pr/ · gh-comment-pr/ memory/ memory-write/ · memory-summarize/ architecture/ arch-solid/ · arch-ddd-review/ security/ sec-audit/ · sec-auth-review/ contracts/ skill-contract.md · memory-contract.md prompt-contract.md · verification-contract.md project-boundaries.md scripts/ loop-back.sh · quality-gate.sh · skill-lint.sh examples/ skill-example.md · agent-flow-example.md

Skills are domain-first, not mechanism-first. react/implement not browser/automate. A React review agent thinks "I need react/review," not "I need browser/verify." Harness capabilities (browser automation, terminal inspection) are internal implementation, not top-level skill families.

16The Core Operating Loop

Ingest, query, and lint are the heartbeat of Sanctum. All three are Bifrost skills. All three write to Sanctum.

wiki/ingest
Read the source document
Write summary page to 10 Knowledge/
Update 01 Indexes/index.md
Update relevant concept + entity pages
Append entry to 01 Indexes/log.md
Open draft PR for review
One source may touch 10–15 wiki pages. This is the compounding operation.
wiki/query
Read index.md to find relevant pages
Load the most relevant pages
Synthesise answer with citations
If answer is valuable: file as new wiki page
Append entry to log.md
Good answers become wiki pages. The wiki gets richer with every question asked.
wiki/lint
Scan for contradictions between pages
Find stale claims newer sources superseded
Identify orphan pages with no inbound links
Flag concepts lacking their own page
Write findings to 00 Inbox/
Run periodically. Lint is what keeps the wiki healthy as it grows.
14Loop-back Gate

~50 lines of shell or Node.js. Everything needed already exists in FODMAPP. The wrapper is the missing connective tissue.

agent
Produces files. Commits to bg/* or codex/* branch. Does not push or open a PR yet.
gate
Loop-back runs quality-gate.sh --llm-only. Captures exit code + ESLint JSON.
→ Exit 0: proceed to PR. → Non-zero: build feedback object, re-invoke agent (max 3×).
→ On exhaustion: write failure log to Sanctum/00 Inbox/ + Slack notification.
gate
Exit 0 confirmed. Loop-back invokes github/open-pr as draft. Branch pushed. PR description includes gate iteration count.
CI
quality-gate.sh --full on PR open. lint:llm:ci as hard failure. Source of truth.
you
Judgment gate. Deterministic checks already passed. Review for correctness, architectural fit, intent match.
